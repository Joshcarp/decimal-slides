Decimal Survey Results Summary

Total respondents: 100

What data types do you use today when you work with money?

  • 29 - shopspring.Decimal
  • 26 - float32/float64
  • 15 - int64 (yen, pennies, or millionth of dollar) or custom struct
  • 8 - big.Int
  • 7 - crdb/apd.Decimal
  • 6 - big.Rat
  • 5 - other decimal/money package
  • 4 - big.Float

While 41 respondents use a base-ten decimal package for representing money, 30 still use binary floating points. 23 use a raw integer or custom package that wraps an integer. 6 use big.Rat.

What data types do you use today when you work with non-money decimals?

  • 66 - float32/float64
  • 18 - A big decimal external package
  • 13 - big.Int
  • 9 - big.Float
  • 5 - Other (int64, json.Number, custom)
  • 4 - big.Rat

What databases / persistence systems do you use when you work with money or decimals

  • 53 - Postgresql
  • 30 - MySQL
  • 11 - MongoDB
  • 9 - SQL Server
  • 4 - Oracle
  • 19 - Other (MS Access, Firebird, SQLite, SAP, DB2, Spanner, …)

What field does your application service?

  • 46 - Finance / E-Commerce
  • 54 - Other  (Education, Government, Healthcare, Cryptocurrency, Advertising, Engineering, Travel, …)

Respondent Comments

  • 3 - I want a built-in decimal type.
  • 2 - Want a money unit (decimal + currency).
  • 1 - Hard to use big.Rat for decimal type (in the context of creating a database/sql driver).

Notes

  • Many people appear to use floats when handling money with Go.
  • The most popular decimal package (shopspring.Decimal) isn't getting a lot of attention these days (shopspring was purchased by another company).
  • There should probably be two decimal value types:
  • An arbitrary precision decimal package that operates like *big.Int.
  • https://github.com/cockroachdb/apd/
  • https://github.com/ericlagergren/decimal/
  • https://github.com/shopspring/decimal
  • A fixed precision decimal package without any pointers in or on the type.
  • https://github.com/robaho/fixed
  • https://github.com/dimdin/decimal

Action Items

  • Continue work on the decimal decomposer https://golang.org/issue/30870 experiment.
  • Ensure this works in a variety of databases (SQL Server, Postgresql, MongoDB, MySQL, Oracle)
  • Ensure the database/sql implementation is sufficient https://golang.org/cl/174181
  • Look into other areas where a decimal type would be useful:
  • https://godoc.org/golang.org/x/text/currency#Amount
  • Look into the possibility of establishing two additional well known packages (fixed, arbitrary precision) with a pool of people willing to help maintain them.
  • If possible, separate out the type from operations.
  • decimal operators could take a DecimalComposer interface, with a type assertion for the well known type at the start of each operation.
